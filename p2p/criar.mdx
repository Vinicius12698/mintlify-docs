## Endpoint

<CodeGroup>

```bash Requisição
POST /p2p
Authorization: Bearer <token>
```

</CodeGroup>

## Descrição

Cria um novo usuário P2P no sistema. Esta é a mesma rota usada para criar testes P2P, mas com `isTrial: false` para usuários pagos.

### Pacotes Disponíveis

- **5da33ae32a9fdb2c33321cca** - PLANO COMPLETO SEM ADULTOS
- **5da17892133a1d61888029aa** - PLANO COMPLETO
- **5e5c14d813d2214122f081df** - PLANO COMPLETO SEM INTER
- **5f8dd931978ef40abef4ab92** - PLANO COMPLETO SEM INTER
- **62000f13b31e82f77f2cc24e** - PLANO CANAIS / SEM VODS
- **627aba573e33db98a64aed77** - PLANO PRINCIPAIS JOGOS
- **67b4b8650f5b7754163fb1f3** - Completo (Sem 24h)



## Parâmetros

<ParamField header="Authorization" type="string" required>
  Bearer token para autenticação
</ParamField>

<ParamField body="isTrial" type="boolean" required>
  Se é um teste gratuito
</ParamField>

<ParamField body="month" type="number" required>
  Duração em meses
</ParamField>

<ParamField body="notes" type="string">
  Observações
</ParamField>

<ParamField body="packageId" type="string" required>
  ID do pacote P2P está disponível na página principal do endpoint
</ParamField>

<ParamField body="passline" type="string" required>
  Senha de linha
</ParamField>

<ParamField body="sale_value" type="number">
  Valor da venda
</ParamField>

<ParamField body="typeUser" type="integer" required>
  Tipo do usuário
</ParamField>

<ParamField body="userline" type="string" required>
  Linha do usuário
</ParamField>

<ParamField body="whatsapp" type="number" >
  WhatsApp do usuário (apenas números)
</ParamField>

<ResponseField name="data" type="object">
